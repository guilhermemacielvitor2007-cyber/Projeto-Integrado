Contexto

Por que a VitaClínica precisa de um BCP/DRP?

Situação antes do projeto

  • Sem nenhum backup configurado
  • Sem procedimento de resposta a incidentes
  • Sem responsáveis definidos para emergências
  • 200 pacientes/dia — qualquer parada causa impacto imediato
  • Dados de saúde = dados sensíveis sob a LGPD

O que este projeto entrega

  • Backup 3-2-1 com criptografia AES-256
  • Procedimentos detalhados para 3 cenários de desastre
  • RPO e RTO definidos para 6 ativos críticos
  • Conformidade com LGPD, ABNT e NIST

72h

Empresa Fictícia

VitaClínica — Clínica Médica de Médio Porte

Unidade Central (sede)

Servidor principal, sala de servidores, NAS local. Atendimento: consultas, exames e laudos.

Unidade Bairro

Conectada via VPN OpenVPN. Dependente do servidor central para PEP e banco de dados de pacientes.

Ativos de TI Críticos

A01 PEPA02 BD Pacientes A03 AgendamentoA04 Financeiro A05 Laudos DICOMA06 VPN

Obrigações Legais

  • LGPD art. 46 — segurança de dados sensíveis de saúde
  • LGPD art. 48 — notificação ANPD em até 72h
  • CFM Res. 2.299/2021 — prontuário guardado por 20 anos
  • ABNT NBR ISO/IEC 27001 — controles de continuidade

Ferramentas Utilizadas (conceitual)

  • Veeam Backup — gestão de backups
  • Zabbix — monitoramento e alertas
  • OpenVPN — VPN entre unidades
  • Amazon S3 sa-east-1 — nuvem
  • rsync + openssl — scripts de backup

RPO — Recovery Point Objective

Máximo de dados que podem ser perdidos. Define a frequência do backup incremental.

RTO — Recovery Time Objective

Tempo máximo tolerado para restaurar o sistema. Define a urgência da resposta ao incidente.

Proteções de Integridade

  • Criptografia AES-256-CBC — chave em local separado dos backups
  • Checksum SHA-256 gerado automaticamente a cada backup
  • Teste mensal automatizado: teste_restauracao.sh (cron 1º dia/mês)

Política de Retenção

  • Dados de saúde (PEP, BD, Laudos): 90 dias
  • Financeiro: 365 dias
  • Agendamento: 60 dias
  • Arquivo DICOM: 20 anos (CFM)
